# 디자인 가이드 (한국어)

## 목적
서비스 페이지와 관리자 페이지에서 일관된 UI 기준을 제공한다.

## 언어 규칙
- 구현 기준 원문은 `docs/design-guide.md`(영문)이다.
- 영문 가이드 변경 시 `docs/design-guide.ko.md`를 반드시 함께 수정한다.

## 디자인 원칙
- 모바일 우선 레이아웃, 데스크톱 웹 호환.
- 핵심 행동을 명확히 노출: 로그인, 빙고 시작, 키워드 전송.
- 빙고의 재미 요소를 유지하되 가독성을 최우선으로 한다.

## 브랜드 방향
- 기본 팔레트는 확정된 시안의 민트/그린 계열을 사용한다.
- 팀 대결 맥락에서만 레드/블루 포인트를 사용한다.
- 텍스트와 인터랙션 요소 대비를 충분히 확보한다.

## 타이포그래피
- 제목 스타일 1종, 본문 스타일 1종을 일관되게 사용한다.
- 제목은 고대비, 고가독성으로 유지한다.
- 본문은 모바일 소형 화면에서도 읽기 쉬워야 한다.

## 간격 및 라운드
- 카드/섹션 간 간격 스케일을 통일한다.
- 카드/버튼의 라운드 스타일을 통일한다.
- 한 화면에 과도한 라운드 값 혼용을 피한다.

## 컴포넌트 규칙
- 로그인 카드: 이름, 비밀번호, 동의 체크, 주요 CTA.
- 키워드 선택: 선택/비선택 상태가 명확한 칩 UI.
- 게임 액션: 키워드 입력 + 보내기 버튼을 한 줄 구성.
- 진행 카드: 완료율, 카운터, 진행 바.
- 빙고 보드: 고정 그리드와 안정적인 셀 크기.
- 팀 패널: 대결 상태, 목표 진행, 남은 줄 수.
- 상호작용 리스트: 내가 준 사람/나에게 보낸 사람 라벨 명확화.

## 반응형 규칙
- 필수 검증 뷰포트:
- 모바일 웹: 375x812 기준
- 데스크톱 웹: 1280px+ 기준
- 레이아웃 겹침/잘림/가로 스크롤이 없어야 한다.

## 접근성
- 텍스트 대비를 충분히 확보한다.
- 모바일 터치 영역 크기를 확보한다.
- 웹 키보드 탐색용 focus-visible 스타일을 제공한다.

## QA 핸드오프
- UI 변경 PR에 가이드 준수 노트를 포함한다.
- 의도적 예외가 있다면 사유를 명시한다.
